# Zero-Hopf and Hopf-Hopf branch switching

Fork computes detailed local normal forms at refined equilibrium codimension-two points and uses
them to construct corrected seeds for every supported outgoing curve. The implementation follows
the multilinear-form conventions used by Kuznetsov and BifurcationKit:

$$
B = D^2F, \qquad C = D^3F.
$$

No factorial is hidden in either form. Complex left eigenvectors use the Hermitian normalization
$$p^*q=1$$, while the Zero-Hopf real mode uses $$p_0^Tq_0=1$$.

## Zero-Hopf

The retained coefficients include $$G_{200}$$, $$G_{011}$$, $$G_{110}$$, $$G_{111}$$,
$$G_{021}$$, the second-order center-manifold vectors, and the two-parameter unfolding directions.
The NS-of-cycles existence test is

$$
\operatorname{Re}(G_{110})G_{011}<0.
$$

When it holds, Fork constructs the second-order periodic orbit and parameter predictor, samples it
on the requested Gauss collocation mesh, and corrects it with `NSCurveProblem`. Fold and Hopf
switches use both signs of the corresponding target defining-system tangent and are corrected on
`FoldCurveProblem` and `HopfCurveProblem`.

## Hopf-Hopf

The nonresonant normal form retains both modal pairs, $$G_{2100}$$, $$G_{0021}$$,
$$G_{1110}$$, $$G_{1011}$$, the complex parameter unfolding matrix, and both NS parameter/frequency
predictors. Fork returns:

- both orientations of Hopf mode 1;
- both orientations of Hopf mode 2;
- the NS curve of periodic orbits generated by mode 1;
- the NS curve of periodic orbits generated by mode 2.

The source frequency is passed explicitly to `HopfCurveProblem`; this is essential because a global
spectral estimator cannot distinguish the two critical pairs at a Hopf-Hopf point.

## Validation and metadata

Every serializable normal form includes the Jacobian and unfolding condition estimates, minimum
left/right eigenvector pairing, maximum eigenvector residual, maximum homological-equation
residual, and low-order resonance distance. Refined curve markers retain the scalar coefficients
and diagnostics needed to audit the classification.

The analytic oracle tests use canonical Zero-Hopf and decoupled cubic Hopf-Hopf unfoldings. They
verify coefficient values and parameter predictions, correct every seed on its target defining
system, and continue at least three target-curve steps.

The nonresonant Hopf-Hopf formulas deliberately reject 1:1 and 1:2 internal resonances. Those points
require separate resonant normal forms and predictors; Fork does not silently reuse the generic
formulas.

## WASM contract

`WasmSystem` exposes four entrypoints:

- `compute_zero_hopf_normal_form` returns only the detailed coefficient object;
- `compute_hopf_hopf_normal_form` returns only the detailed coefficient object;
- `switch_from_zero_hopf` returns the normal form, four equilibrium-curve seeds, and the optional
  NS-cycle seed;
- `switch_from_hopf_hopf` returns the normal form, four Hopf-curve seeds, and both NS-cycle seeds.

Every returned `Codim2BranchSeed` contains predictor/corrector residuals and correction iterations.
The cycle seeds additionally retain period, mesh dimensions, and the NS multiplier cosine needed to
start the existing curve runner without reconstructing hidden normal-form state.
